# richman6 MVP 编辑器配置清单

这个 MVP 已经把核心玩法闭环放进代码里：`回合推进 → 掷骰移动 → 买地收租 → 基础卡牌干预 → 胜负判定`。

## 现在代码就能完成的部分

- 自动初始化 2 人对局
- 自动推进回合与骰子结算
- 32 格单环地图的数据表驱动规则
- 基础地产购买、升级、收租
- 10 张 MVP 卡牌中的自动触发逻辑
- 破产判负与回合上限资产结算
- 通过日志验证玩法成立

## 仍需手动完成的编辑器配置

- 在地图里创建并保存目标 NodeGraph，ID 对应 `src/config.ts:1`
- 搭一张测试地图，至少保留 1 条闭环路线与 32 个格子位
- 准备“格子索引 -> 场景表现”的映射方案，保证后续能把代码状态映射到地图实体
- 配置玩家出生点与基础移动路线
- 如果要把商店格做成交互式玩法，需要补齐货币、背包、商店模板与商店组件

## 当前默认假设

- 首版使用“格子索引 + 数据表”驱动，不依赖复杂地图识别
- 商店格先走自动结算，不依赖 UI 或按钮
- 分岔、传送、复杂封路和全局计时器均未启用
- 角色系统、辅助单位系统、表现层资源仍留在后续阶段
